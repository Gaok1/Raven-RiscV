Architecture

The classic pipeline, made visible

Falcon makes every stage of instruction processing observable and approachable.

01

Fetch

The program counter points to the next 32-bit instruction word. Falcon reads it from memory and advances PC by 4.

Programs

Real programs, real assembly

Explore the included example programs — from hello world to recursive algorithms.

fib.fas RISC-V Assembly 
; Compute fib(6) recursively
li   a0, 6
call fib
call end

fib:
    li   t0, 1
    beq  a0, t0, one
    beq  a0, zero, zero_label
        push ra               ; save return address
        addi a0, a0, -1       ; n-1
        push a0               ; push n-1
        call fib              ; fib(n-1)
        pop  a0               ; pop n-1
        push s0               ; push fib(n-1)
        addi a0, a0, -1
        call fib              ; fib(n-2)
        pop  s1               ; s1 = fib(n-1)
        add  s0, s0, s1       ; combine results
        pop  ra               ; restore return address
        ret
    one:
        li   s0, 1
        ret
    zero_label:
        li   s0, 0
        ret
end:
    li   a0, 0
    li   a7, 93      ; syscall: exit
    ecall

Recursive Fibonacci

Demonstrates recursive function calls using push/pop pseudo-instructions that manage the stack automatically, plus call/ret for proper calling convention.

  • Stack management with push / pop
  • Recursive call and ret
  • Conditional branches (beq)
  • Register conventions (a0, s0, ra)
bubble_sort_20.fas RISC-V Assembly 
.data
arr:  .word 20, 1, 19, 2, 18, 3, 17, 4, 16, 5, 15, 6, 14, 7, 13, 8, 12, 9, 11, 10
msg:  .asciz "Sorted values:"
nl:   .byte 10, 0

.text
    printStrLn msg

    la   t0, arr      ; base address
    li   t4, 20       ; n = 20
    li   t1, 0        ; i = 0

outer:
    li   t2, 0
    sub  s2, t4, t1
    addi s2, s2, -1

inner:
    slli t3, t2, 2   ; offset = j * 4
    add  t3, t0, t3  ; &arr[j]
    addi t5, t3, 4   ; &arr[j+1]
    lw   s0, 0(t3)
    lw   s1, 0(t5)
    blt  s0, s1, no_swap
    sw   s1, 0(t3)
    sw   s0, 0(t5)
no_swap:
    addi t2, t2, 1
    blt  t2, s2, inner
    addi t1, t1, 1
    blt  t1, t4, outer

    li   t2, 0
print_loop:
    slli t3, t2, 2
    add  t3, t0, t3
    lw   a0, 0(t3)
    print a0
    printStr nl
    addi t2, t2, 1
    blt  t2, t4, print_loop

    li   a0, 0
    li   a7, 93      ; syscall: exit
    ecall

syscall_echo_linux.fas RISC-V Assembly 
.data
buf: .space 64

.text
    li   a0, 0       ; fd = stdin
    la   a1, buf
    li   a2, 64      ; max bytes
    li   a7, 63      ; syscall: read
    ecall           ; a0 = bytes read

    mv   t0, a0      ; n = bytes_read
    li   a0, 1       ; fd = stdout
    la   a1, buf
    mv   a2, t0
    li   a7, 64      ; syscall: write
    ecall

    li   a0, 0
    li   a7, 93      ; syscall: exit
    ecall

Syscall I/O — Echo

Raw Linux-style ABI: a7 holds the syscall number, a0–a5 are arguments. Reads from stdin, writes to stdout. Shows how high-level I/O maps directly to hardware.

Get Started

Up and running in seconds

Falcon ships as a prebuilt binary — or you can build it yourself with Cargo.

Recommended

Download & Run

Grab the latest release for your platform and run it directly — no toolchain needed.

1
Download the binary for your OS from the Releases page.
2
Run the executable — Falcon's IDE opens immediately.
3
Write your first program and press Ctrl+R to assemble.

Build from Source

Requires the Rust toolchain. Gives you the latest code and lets you embed Falcon in your own projects.

$ git clone <repo-url> falcon-asm
$ cd falcon-asm
$ cargo run
✓ Falcon IDE launched

Embed the Assembler

Use Falcon's assembler programmatically inside your own Rust projects.

use falcon::program::{load_words, load_bytes, zero_bytes};

let prog = falcon::asm::assemble(source, base_pc)?;
load_words(&mut mem, base_pc, &prog.text)?;
load_bytes(&mut mem, prog.data_base, &prog.data)?;
let bss_base = prog.data_base + prog.data.len() as u32;
zero_bytes(&mut mem, bss_base, prog.bss_size)?;
Who is it for?

Built for learners, teachers, and tinkerers

Whether you're a student tackling your first computer architecture course, a teacher looking for a zero-friction demo tool, or a hobbyist curious about how CPUs really work — Falcon gives you a safe, observable playground.

  • 🎓
    Students — Follow textbooks exactly. Falcon's register naming and little-endian layout mirror standard RISC-V courses.
  • 🖥️
    Teachers — Live-demo every pipeline stage. Assemble student assignments and step through them in class.
  • ⚙️
    Hobbyists — Explore the RV32I ISA without reading thousands of lines of optimization-heavy code.
